# 潮汐孤岛（CCGame）

一个基于 **Vanilla JavaScript + Vite** 的 2D 生存建造网页游戏。玩家会出生在一片程序生成的群岛中，白天采集资源、制作工具和建造设施，夜晚抵御从海岸出现的敌人，尽可能延长生存天数并提高分数。

## 项目特点

- **动森风格 UI**：参考 [animal-island-ui](https://github.com/guokaigdg/animal-island-ui) 设计语言（薄荷青绿主色、温暖大地色文字、奶油米白背景、pill 圆角与游戏按键 3D 立体阴影、Nunito + Noto Sans SC + Zen Maru Gothic 字体）重构界面。
- **桌面 & 移动端通用**：自动识别触屏环境，提供虚拟摇杆、攻击/交互/冲刺按钮、左右面板抽屉、背包长按菜单等触屏交互。桌面端支持 `M`/`B`/`C` 快捷键随时显示或隐藏海图、背包、制作面板。
- **轻量音效**：通过 Web Audio API 现场合成短音，无需额外资源；覆盖采集、攻击、击杀、建造、烹饪、取水、进食、钓鱼、受伤、UI 交互等关键操作。
- **程序化群岛地图**：使用噪声与岛屿描述符生成大世界，并按区块动态加载。
- **生存循环完整**：包含生命、饥饿、口渴、体力、昼夜变化与天气提示。
- **采集 / 制作 / 建造**：可采集木材、石块、纤维、浆果、椰子等资源，制作石斧、石镐、长矛、鱼竿、篝火、木墙、木地板、雨水收集器、混合种子包和种植箱。
- **战斗与防守**：夜晚会出现“暗潮爬行者”，需要通过武器和营地布局进行防守。
- **交互式 UI**：包含背包、制作面板、海图、世界目标面板、右键物品菜单和提示文本。
- **钓鱼系统**：可对近处水面抛竿，在咬钩窗口内收线获得不同鱼类。
- **种植 / 烹饪**：可在种植箱中培育南瓜，并在篝火旁把鱼获烤成熟食。

## 当前玩法

1. 开始游戏后先在岛上采集树木、岩石、灌木和棕榈。
2. 通过右侧制作面板合成基础工具与建造套件。
3. 左键切换当前手持物，靠近目标后进行采集、攻击、建造或使用。
4. 使用篝火补充夜间营地收益，利用木墙、地板和种植箱搭建营地。
5. 建造雨水收集器补充饮水，或通过鱼竿在浅水 / 深水区域捕鱼并烹饪成烤鱼。
6. 把浆果和纤维制成混合种子包，在种植箱中培育南瓜补充食物。
7. 在夜晚击退敌人，尽可能维持角色状态并提高生存评分。

## 操作说明

### 桌面端

- `W / A / S / D`：移动
- `Shift`：冲刺
- `鼠标左键`：行动 / 攻击 / 使用当前手持
- `鼠标右键（场景）`：取消当前手持
- `E`：与当前目标交互
- `M`：显示 / 隐藏 海图面板
- `B`：显示 / 隐藏 背包面板
- `C`：显示 / 隐藏 制作面板
- `背包左键`：切换当前手持物品
- `背包右键`：打开物品菜单
- `Esc`：关闭物品菜单

### 移动端 / 触屏

- **左下虚拟摇杆**：移动角色（支持斜向）
- **红色按钮**：攻击 / 使用当前手持（按住可连续触发）
- **青色按钮**：与当前目标交互
- **黄色按钮**：按住冲刺
- **顶部圆形按钮**：分别切换“状态/海图”与“背包/制作”面板
- **轻点画布**：设置瞄准方向并触发一次主动作
- **拖动画布**：调整瞄准方向（不会触发动作）
- **背包格长按**：打开物品菜单

## 本地运行

### 环境要求

- Node.js 20.x (与仓库当前 GitHub Pages 工作流保持一致)
- npm

### 安装依赖

```bash
npm ci
```

### 启动开发环境

```bash
npm run dev
```

启动后按 Vite 默认提示访问本地地址即可。

### 构建生产版本

```bash
npm run build
```

构建产物输出到 `dist/` 目录。

## 联机功能（实验性）

基于 **WebRTC + Trystero** 实现的零后端联机：使用公共 BitTorrent tracker 作为 WebRTC 信令通道，不需要自建服务器。

### 使用方法

1. 在开始界面点击右下角「联机」按钮打开房间面板。
2. 填写昵称（仅在本地保存）。
3. **创建房间**：点击「创建房间」会自动生成 6 位房间码。把房间码分享给好友。
4. **加入房间**：在输入框中粘贴房间码，点击「加入房间」。
5. 进入房间后可在面板内文字聊天、查看成员列表、显示与各 peer 的实时延迟。
6. 点击「离开房间」断开连接。

### 当前能力与后续计划

- **当前版本**：仅实现联机基础设施（房间创建/加入、成员同步、聊天）。游戏世界本身仍为单机权威，多玩家世界同步将在后续版本中提供。
- **后续步骤**：玩家状态多实例化 → Host 权威的输入/移动同步 → 世界对象与昼夜增量同步 → 行为请求-确认 → 多玩家背包与合成。

### 网络说明

- 默认使用公共 STUN（Google / Cloudflare）。绝大多数家庭网络可直接 P2P 互联。
- 在严格 NAT 环境下 WebRTC 可能无法穿透；遇到连接失败时可换用其他网络或加入自带 TURN（后续 UI 选项）。
- 信令依赖公共 BitTorrent tracker，部分受限网络可能屏蔽，可考虑切换 strategy（Nostr / MQTT，后续可选）。

## 项目结构

```text
CCGame/
├─ index.html              # 页面结构与 HUD / 弹层
├─ styles/                 # 全局样式
├─ src/
│  ├─ app/                 # 启动、输入绑定、主循环、移动端控件
│  ├─ components/          # 地块、资源、敌人、结构定义
│  ├─ core/                # 常量、状态、DOM 引用
│  ├─ data/                # 物品与配方数据
│  ├─ ecs/                 # 简单 ECS 核心
│  ├─ inventory/           # 背包与物品操作
│  ├─ render/              # 场景绘制与特效
│  ├─ systems/             # 行为、运行时与工厂系统
│  ├─ ui/                  # 面板、菜单、提示交互
│  ├─ world/               # 地图生成与世界辅助逻辑
│  ├─ net/                 # 联机：Trystero 传输、协议、会话
│  └─ workers/             # 区块生成 Worker
├─ vite.config.mjs         # Vite 配置
└─ .github/workflows/      # GitHub Pages 部署流程
```

## 技术说明

- 使用原生 Canvas 进行渲染。
- 通过模块拆分组织 ECS、渲染、UI、世界生成与物品系统。
- 使用 Web Worker 处理世界区块生成，降低主线程压力。
- 使用 Vite 进行开发与构建，`base: './'` 方便静态部署。

## 部署

仓库已包含 GitHub Pages 工作流：

- 推送到 `main` 分支后自动执行 `npm ci` 与 `npm run build`
- 构建完成后发布 `dist/` 到 GitHub Pages
